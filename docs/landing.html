Why Privacy Matters for Dictation

Dictation captures what you think. Your voice contains:

  • Passwords you speak to unlock devices
  • Confidential business strategy and client names
  • Medical conditions and personal information
  • Personal messages to friends and family
  • Draft email you never send (the thinking process)

Sending this to a cloud service—even with encryption—creates a permanent record. Sotto keeps everything on your Mac. You own your voice.

Verify the "Zero Network" Claim

Don't trust us. Verify it yourself using standard macOS tools:

  1. Little Snitch: Install Little Snitch, run Sotto, and watch the network monitor. No outbound connections (except the one optional GitHub check for updates).
  2. Console.app: Check /var/log/system.log while dictating. No DNS queries for cloud services.
  3. Terminal (nettop): Run nettop -n and filter by Sotto. See zero packets sent to external servers.
  4. Source code: Read the entire codebase on GitHub. Search for "http", "https", "URLSession". In the default build the only network call is the explicit "Check for Updates" in UpdateChecker.swift. Optional model providers (Ollama, and bring-your-own-key cloud) live in one file — Providers.swift — and are off by default: with no provider selected, none of that code is ever constructed or reachable, so the default build stays network-silent and the Little Snitch test above holds.
